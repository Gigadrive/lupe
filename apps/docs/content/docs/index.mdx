**lupe** (German for *magnifying glass*) is a platform- and provider-agnostic AI code review
agent. It reviews your pull requests — or a local diff — using **your own** provider tokens, as an
open-source alternative to hosted tools like CodeRabbit, Cursor Bugbot, Greptile, and Macroscope.

There is no SaaS to sign up for and no per-seat pricing: you point lupe at a model you already pay
for (Anthropic, OpenAI, Google, Amazon Bedrock, or any OpenAI-compatible gateway) and it does the
review on your infrastructure and your budget.

## Why lupe

- **Provider-agnostic.** One review engine, any model. Switching provider or model is a single
  config value — the engine asks for a *task* (triage, review, verify, deep), never a hard-coded
  model id.
- **Bring your own tokens.** Your keys, your spend. The only secret a repository needs is your
  provider API key; GitHub access uses the built-in `GITHUB_TOKEN`.
- **Two surfaces, one core.** A reusable **GitHub Action** reviews pull requests, and a local
  **CLI** reviews diffs from your terminal — both call the exact same engine. An embeddable
  **SDK** lets you run reviews from your own code.
- **Precision-first.** Generation is deliberately high-recall, then a grounding verifier and a
  filter chain gate what actually gets published — the goal is a handful of actionable comments per
  PR, not noise.
